## Summary

Package the PaysafeMigrationOracle into a reproducible, minimal Docker image that starts the MCP SSE server and the Streamlit UI as co-located processes under a POSIX sh supervisor. The image uses a two-stage build: a builder stage installs dependencies with `uv` (sourced from the official uv image) and pre-bakes the sentence-transformers model; the final stage copies only the virtualenv, model cache, and application source into a `python:3.11-slim` base running as a non-root user (`oracle`, uid 1000).

---

## Technical Context

**Language/Version**: Python 3.11 (`python:3.11-slim` base image)

**Primary Dependencies**: `mcp` (FastMCP / SSE transport), `streamlit`, `sentence-transformers`, `torch` (CPU), `neo4j`, `langchain`, `uv` (build-time only)

**Storage**: Neo4j 5 (external sidecar via compose); `/data` volume mount for runtime config injection; `/app/.cache/huggingface` for pre-baked model (baked into image)

**Testing**: `pytest` (dev-only, excluded from image via `--no-dev`); Dockerfile correctness verified by `docker build` exit code; model pre-bake verified by `RUN` step failing on download error

**Target Platform**: `linux/amd64`; `python:3.11-slim` (Debian bookworm slim — glibc required for sentence-transformers)

**Project Type**: Deployment artifact (Dockerfile + compose + entrypoint script)

**Performance Goals**: Container ready (both endpoints responding) within 60 s of start; first query latency ≤ subsequent query latency (no cold model download); image ≤ 3 GB compressed

**Constraints**: Non-root runtime (uid 1000); no secrets in image layers; POSIX sh entrypoint (no bash-isms); reproducible build (pinned uv, `uv.lock --frozen`); fail-fast on either process exit

**Scale/Scope**: Single-container deployment; two co-located processes; one external dependency (Neo4j)

## Multi-Stage Dockerfile Design

### Stage 1 — `uv-bin` (uv binary source)

```
FROM ghcr.io/astral-sh/uv:0.5 AS uv-bin
```
Single-purpose: provides the `uv` binary via `COPY --from`. Not derived from — just used as a source.

### Stage 2 — `builder`

Base: `python:3.11-slim`

Layer ordering (cache-stable across source-only edits):

| Order | Instruction | Cache invalidated when |
|-------|-------------|----------------------|
| 1 | `COPY --from=uv-bin /uv /usr/local/bin/uv` | uv version tag changes |
| 2 | `RUN apt-get install -y --no-install-recommends gcc && rm -rf /var/lib/apt/lists/*` | Base image changes |
| 3 | `WORKDIR /app` | Never (constant) |
| 4 | `ENV HF_HOME=/app/.cache/huggingface` | Never (constant) |
| 5 | `COPY pyproject.toml uv.lock ./` | Lock file changes |
| 6 | `RUN uv sync --frozen --no-dev` | Lock file changes (re-downloads all deps including torch) |
| 7 | `ARG SENTENCE_TRANSFORMERS_MODEL=all-mpnet-base-v2` + `RUN python -c "from sentence_transformers import SentenceTransformer; SentenceTransformer('${SENTENCE_TRANSFORMERS_MODEL}')"` | Model ARG changes |
| 8 | `COPY migration_oracle/ ./migration_oracle/` | Source code changes only |

**Key invariant**: Steps 6 and 7 (the two heavy layers — pytorch install and model download) are stable as long as `uv.lock` and the model ARG are unchanged. Source edits (step 8) only invalidate step 8.

**WORKDIR + HF_HOME must precede uv sync** (steps 3–4 before step 5): `WORKDIR /app` establishes the directory under which `.venv/` and `.cache/` are created, so that the paths `COPY --from=builder /app/.venv` and `COPY --from=builder /app/.cache` in the final stage resolve to the same absolute locations. `ENV HF_HOME=/app/.cache/huggingface` must be set before `uv sync` so that any eager huggingface cache writes during install also land under `/app/.cache`.

**FR-014 enforcement**: The `RUN` in step 7 uses a plain Python one-liner that raises an exception (non-zero exit) if the download fails. No `|| true` or output redirection is permitted.

### Stage 3 — `final`

Base: `python:3.11-slim`

```
FROM python:3.11-slim AS final

# Re-declare ARG so ${SENTENCE_TRANSFORMERS_MODEL} resolves in the ENV below.
# ARG values do not propagate between stages — omitting this silently bakes
# ENV SENTENCE_TRANSFORMERS_MODEL= (empty string) into the image.
ARG SENTENCE_TRANSFORMERS_MODEL=all-mpnet-base-v2

# System: curl for health-check, procps for pgrep/pkill (fail-fast validation)
RUN apt-get update && apt-get install -y --no-install-recommends curl procps \
    && rm -rf /var/lib/apt/lists/*

# Non-root user
RUN addgroup --system --gid 1000 oracle \
    && adduser --system --uid 1000 --gid 1000 --no-create-home oracle

# Copy virtualenv and model cache from builder
COPY --from=builder /app/.venv /app/.venv
COPY --from=builder /app/.cache /app/.cache

# Copy application source
WORKDIR /app
COPY migration_oracle/ ./migration_oracle/

# Copy entrypoint
COPY docker/entrypoint.sh ./entrypoint.sh
RUN chmod +x ./entrypoint.sh

# Data volume and ownership
RUN mkdir -p /data && chown -R oracle:oracle /app /data

# Environment
ENV PYTHONUNBUFFERED=1 \
    MCP_TRANSPORT=sse \
    MCP_HOST=0.0.0.0 \
    MCP_PORT=8080 \
    STREAMLIT_SERVER_PORT=8501 \
    HF_HOME=/app/.cache/huggingface \
    SENTENCE_TRANSFORMERS_MODEL=${SENTENCE_TRANSFORMERS_MODEL} \
    PATH="/app/.venv/bin:$PATH"

# Ports
EXPOSE 8080 8501

# Volume
VOLUME /data

# Health-check (both services)
HEALTHCHECK --interval=30s --timeout=10s --start-period=60s --retries=3 \
  CMD curl -fsS --max-time 5 http://127.0.0.1:${MCP_PORT:-8080}/sse; ret=$?; \
      ([ $ret -eq 0 ] || [ $ret -eq 28 ]) && \
      curl -fsS --max-time 5 http://127.0.0.1:${STREAMLIT_SERVER_PORT:-8501}/_stcore/health

USER oracle

ENTRYPOINT ["/app/entrypoint.sh"]
```

## Entrypoint Script Design (`docker/entrypoint.sh`)

```sh
#!/bin/sh
set -e

# Start MCP SSE server in background
python -m migration_oracle.mcp.server &
MCP_PID=$!

# Start Streamlit UI in background
streamlit run migration_oracle/streamlit_app/app.py \
    --server.headless true \
    --server.port "${STREAMLIT_SERVER_PORT:-8501}" \
    --server.address 0.0.0.0 &
ST_PID=$!

# Exit as soon as either child exits; propagate exit code
wait -n $MCP_PID $ST_PID
EXIT_CODE=$?

# Signal the surviving process to stop
kill $MCP_PID $ST_PID 2>/dev/null || true
wait $MCP_PID $ST_PID 2>/dev/null || true

exit $EXIT_CODE
```

**Notes**:
- `set -e` causes the script to exit immediately on any error before backgrounding.
- `wait -n` is available in `dash` (the `/bin/sh` in `python:3.11-slim`).
- The surviving process is signalled with `kill` and waited for to avoid zombie processes.
- `STDOUT` and `STDERR` from both child processes flow directly to container stdout/stderr — no redirection.

## Model Pre-Bake Strategy

The sentence-transformers model is downloaded during `docker build` in the builder stage. The download command is a plain Python one-liner:

```dockerfile
ARG SENTENCE_TRANSFORMERS_MODEL=all-mpnet-base-v2
ENV HF_HOME=/app/.cache/huggingface
RUN python -c "\
from sentence_transformers import SentenceTransformer; \
SentenceTransformer('${SENTENCE_TRANSFORMERS_MODEL}')" 
```

The model files land in `${HF_HOME}/hub/models--<name>/`. This directory is copied to the final stage:

```dockerfile
COPY --from=builder /app/.cache /app/.cache
```

**FR-014 enforcement**: If the download fails (network unavailable, invalid model name), `SentenceTransformer(...)` raises a Python exception, causing the `RUN` instruction to exit non-zero and abort `docker build`. No fallback or suppression is present.
